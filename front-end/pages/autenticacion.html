Módulo de seguridad

Control de acceso avanzado para el campus digital.

Este módulo establece enlaces entre el front-end y el backend Ruby, administrando la sesión, el token JWT y los roles con el mayor cuidado.

Volver al inicio
JWT + BCrypt

Sesiones protegidas

Valida usuarios, protege rutas y evita acceso genérico con roles segmentados.

Lógica de negocio

Cómo funciona este módulo con el backend

La autenticación no solo verifica credenciales. Comprueba la vigencia del token, el estado de bloqueo de la cuenta y define qué módulos se muestran en la UI.

Funciones principales

  • Inicio de sesión con token seguro.
  • Redirección según rol: alumno, docente, administrativo.
  • Mensajes claros de sesión caducada o acceso denegado.
  • Conexión fluida con los endpoints Ruby que emiten JWT.
